服务器层
Backend Services FastAPI · Python 3.11+ · Uvicorn · pydantic-settings
packages/backend/
① 配置层 (app/config.py)
• pydantic-settings 从 ../../.env 读取全局配置
• API Keys：DEEPSEEK_* / GEMINI_* / BAIDU_*（预留）
• 基础设施：REDIS_URL / CORS_ORIGINS / DEBUG
• 安全 & 限流：API_SECRET_KEY / RATE_LIMIT_PER_MINUTE / FREE_DAILY_*
• @lru_cache 全局单例 get_settings()
② 中间件层 (app/main.py 内注册)
• CORS 跨域中间件：允许 Vercel 域 + chrome-extension://* origins
• Rate Limiting 中间件：滑动窗口算法，per IP，默认 30 req/min，返回 429
• 可选 API Key 认证：设置 API_SECRET_KEY 后，翻译接口需携带 X-API-Key 头
③ 路由层 (app/routers/translate.py)
• prefix: /api/v1
• POST /api/v1/translate — 核心翻译接口
• GET /api/v1/languages — 支持的语言列表
• GET /health — 健康检查
• 错误映射：400 / 502 / 503
④ 服务调度层 (app/services/translator.py — TranslatorService)
Processes → 1 接收请求(多段文本)→ 2 Redis mget 批量查缓存→ 3 找出未命中文本
requests → 4 调翻译引擎处理→ 5 新结果写 Redis (setex, TTL 7d)→ 6 Merge cached + 新结果返回
引擎注册表：engines dict[EngineType → BaseEngine]，新增引擎加一行即可
⑤ 缓存层 (app/services/cache.py)
• TranslateCache 类
• 键格式：trans:<SHA-256>
• TTL: 7 天（86400×7）
• get / set / get_batch（mget）
• 容错：Redis 故障时静默降级
⑥ 引擎层 (app/services/engines/)
BaseEngine（base.py）：抽象基类，定义 translate() 接口规范 · TranslateRequest / TranslateResult 数据模型 · EngineType 枚举 · 文本条数 / 长度校验
DeepSeekEngine
deepseek.py：AsyncOpenAI SDK，[SEP] 批量合并翻译策略
GeminiEngine
gemini.py：google-genai SDK 异步 generate_content，[SEP] 策略
BaiduEngine 待接入
baidu.py：httpx 异步调用百度翻译 REST API，暂未启用
