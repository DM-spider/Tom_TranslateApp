## 登录功能和订阅计划 -- 具体执行步骤

### Phase 1：基础改造（约 3-4 天）

#### 1.1 网页翻译改为手动触发（0.5 天）

**改动范围**：仅前端，改一个文件

- TranslatePanel.tsx：
  - 移除 `handleInputChange` 中的 `debounceTimer` 自动触发逻辑
  - 移除 `useEffect` 中语言/引擎切换时的自动触发（第 114-119 行）
  - 在输入框下方添加一个「翻译」按钮，`onClick` 调用 `triggerTranslation(inputText)`
  - 保留快捷键支持：`Ctrl+Enter` 触发翻译（提升体验）

#### 1.2 接入 LibreTranslate 免费翻译引擎（1-1.5 天）

**后端改动**：

1. **`docker-compose.yml`**：增加 LibreTranslate 服务
   ```yaml
   libretranslate:
     image: libretranslate/libretranslate:latest
     ports:
       - "5000:5000"
     environment:
       - LT_LOAD_ONLY=en,zh,ja,ko,fr,de,es,ru  # 只加载需要的语言包，节省内存
   ```

2. **新增 `engines/libre.py`**：继承 `BaseEngine`，通过 `httpx` 调用 `http://libretranslate:5000/translate`

3. **`engines/base.py`**：`EngineType` 枚举增加 `LIBRE = "libre"`

4. **translator.py**：注册 `LibreTranslateEngine`

5. **config.py**：增加 `libre_translate_url: str = "http://localhost:5000"`

**前端改动**：

6. 引擎列表增加「机器翻译（免费）」选项
7. 默认引擎改为 `libre`（让新用户先体验免费功能）

### Phase 2：用户认证系统（约 3-4 天）

#### 2.1 数据库搭建（0.5 天）

- `docker-compose.yml` 启用注释掉的 PostgreSQL 服务
- 新增 `app/database.py`：SQLAlchemy async 数据库连接
- 新增 `app/models/user.py`：用户表（id, email, password_hash, plan, created_at）
- 新增 `app/models/usage.py`：用量表（user_id, date, llm_count）
- 依赖新增：`sqlalchemy[asyncio]`, `asyncpg`, `python-jose[cryptography]`, `passlib[bcrypt]`

#### 2.2 认证接口（1 天）

新增 `app/routers/auth.py`：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/auth/register` | POST | 邮箱 + 密码注册 |
| `/api/v1/auth/login` | POST | 登录，返回 JWT access_token |
| `/api/v1/auth/me` | GET | 获取当前用户信息 + 今日用量 |

认证方式：**JWT（JSON Web Token）**
- 登录成功返回 `access_token`（有效期 7 天）
- 前端/扩展存储 token，请求时放到 `Authorization: Bearer <token>` 头
- 后端新增 `app/dependencies.py`：`get_current_user()` 依赖注入（可选认证，未登录返回 None）

#### 2.3 翻译接口配额控制（1 天）

改造 translate.py 路由：

```python
@router.post("/translate")
async def translate(req: TranslateRequest, user: User | None = Depends(get_optional_user)):
    # 免费引擎（libre）：任何人都可以用
    if req.engine == EngineType.LIBRE:
        return await translator.translate(req)
    
    # LLM 引擎：需要登录
    if user is None:
        raise HTTPException(403, "使用 AI 翻译需要登录，免费机器翻译无需登录")
    
    # 检查配额
    today_count = await get_today_llm_usage(user.id)
    daily_limit = 999999 if user.plan == "pro" else 10
    if today_count >= daily_limit:
        raise HTTPException(429, f"今日 AI 翻译次数已用完（{daily_limit}次），请使用免费机器翻译或升级 Pro")
    
    result = await translator.translate(req)
    await increment_usage(user.id)
    return result
```

#### 2.4 前端登录 UI（1-1.5 天）

**网页端**（`packages/web`）：
- 新增 `components/AuthModal.tsx`：登录/注册弹窗（邮箱 + 密码）
- 新增 `lib/auth.ts`：token 存储/读取、请求拦截器自动加 Authorization 头
- TranslatePanel.tsx：选择 LLM 引擎时，若未登录弹出登录弹窗；显示剩余次数

**扩展端**（`packages/extension`）：
- `popup/App.tsx` 设置 Tab 增加登录入口
- `utils/settings.ts`：存储 token 到 `chrome.storage.local`
- `utils/api.ts`：请求时自动携带 Authorization 头
- `background.ts`：LLM 翻译请求失败 403 时，通知 content script 显示「需登录」提示

### Phase 3：Pro 订阅（上线后再做）

这部分建议**先不实现**，MVP 阶段用手动管理：
- 用户付费后你在数据库手动把 `plan` 改成 `pro`
- 等有了真实付费用户后，再接入 Stripe / 支付宝做自动化
