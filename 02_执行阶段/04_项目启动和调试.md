### 启动与调试

> 以下所有路径均相对于项目根目录 `Tom_TranslateApp/`。

#### 一、环境准备（仅首次）

```bash
# 1. 安装全部依赖（在项目根目录执行，pnpm workspace 会同时处理所有子包）
pnpm install

# 2. 确认 .env 文件存在于项目根目录，并填入 API Key
#    必须至少配置一个翻译引擎的 Key：
#      DEEPSEEK_API_KEY=sk-xxxx
#      GEMINI_API_KEY=AIzaSyXXXX
```

#### 二、启动后端 API（每次使用前必须启动）

```bash
# 进入后端目录并启动（--reload 开启热重载，修改代码后自动重启）
cd packages/backend
uv run uvicorn app.main:app --reload --port 8000

# 验证后端是否正常运行：浏览器访问 http://localhost:8000/health
# 返回 {"status": "ok"} 即表示成功
# 交互式 API 文档：http://localhost:8000/docs
```

#### 三、构建浏览器扩展

**方式 A：开发模式（推荐调试时使用，支持热重载）**
```bash
cd packages/extension
pnpm dev            # 产物输出到 .output/chrome-mv3-dev/
```

**方式 B：生产构建（测试或发布时使用）**
```bash
cd packages/extension
pnpm build          # 产物输出到 .output/chrome-mv3/
```

#### 四、在 Chrome 中加载扩展

1. 打开 `chrome://extensions/`
2. 右上角开启 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择对应目录：
   - 开发模式选 `packages/extension/.output/chrome-mv3-dev/`
   - 生产构建选 `packages/extension/.output/chrome-mv3/`
5. 加载成功后在工具栏可看到 **Tom Translate** 图标

#### 五、日常使用流程

```
┌──────────────────────────────────────────────────┐
│  终端 1：启动后端                                  │
│  cd packages/backend                              │
│  uv run uvicorn app.main:app --reload --port 8000 │
├──────────────────────────────────────────────────┤
│  终端 2：启动扩展开发服务（可选，仅开发时需要）       │
│  cd packages/extension                            │
│  pnpm dev                                         │
└──────────────────────────────────────────────────┘
```

> **注意事项**：
> - 每次重新构建或更新扩展后，需要在 `chrome://extensions/` 页面点击扩展卡片上的 **刷新按钮** ↻
> - 刷新扩展后，**已打开的网页必须重新加载（F5）**，新的 Content Script 才会注入
> - 后端未启动时，划词翻译和整页翻译都会报错——请确保终端 1 始终在运行

#### 六、快速验证

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 访问 `http://localhost:8000/health` | 返回 `{"status": "ok"}` |
| 2 | 打开任意英文网页，选中一段文字 | 选区右侧出现蓝色「译」按钮 |
| 3 | 点击「译」按钮 | 弹出翻译气泡，显示中文译文 |
| 4 | 点击扩展图标 → 翻译此页 | 右下角出现进度条，页面逐段显示译文 |
| 5 | 按 `Alt+A` | 同上，快捷键触发整页翻译 |
