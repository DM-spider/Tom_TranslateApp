# 项目文件功能介绍

> 本文档说明项目初始化阶段创建的所有文件，每个文件在整个项目中扮演的角色和具体职责。

---

## 根目录文件

### `pnpm-workspace.yaml`

**文件类型**：pnpm 工作区配置

**职责**：声明 Monorepo 中哪些子目录是独立的 npm 包。被声明的包可以互相引用（就像引用发布到 npm 的第三方包一样），并共享 `node_modules`，避免重复安装依赖。

目前声明了三个子包：`packages/web`（网页前端）、`packages/extension`（浏览器扩展）、`shared`（共享类型库）。后端是 Python 项目，不参与此配置。

---

### `package.json`（根目录）

**文件类型**：Node.js 项目描述文件

**职责**：整个 Monorepo 的统一控制台。不包含任何业务代码，只负责两件事：

1. **声明开发依赖**：`concurrently`（同时启动多个终端进程）、`typescript`（全局类型支持）
2. **定义统一脚本**：让开发者无需进入子目录，在项目根目录就能操作所有子项目

| 脚本 | 作用 |
|------|------|
| `pnpm dev` | 同时启动后端 + 网页前端（开发常用） |
| `pnpm dev:web` | 单独启动 Next.js 前端 |
| `pnpm dev:ext` | 单独启动浏览器扩展开发服务 |
| `pnpm dev:backend` | 单独启动 FastAPI 后端 |
| `pnpm build:web` | 构建网页前端（生产版本） |
| `pnpm build:ext` | 构建浏览器扩展（打包用于发布） |

---

### `docker-compose.yml`

**文件类型**：Docker Compose 服务编排配置

**职责**：定义本地开发所需的基础设施服务，用 Docker 容器替代手动安装软件。

目前包含的服务：
- **Redis**：翻译结果缓存，运行在 `localhost:6379`。相同的翻译请求命中缓存后秒回，不消耗 API 额度

PostgreSQL 服务已注释，等项目需要持久化用户账号、翻译历史等数据时再启用。

---

### `.env.example`

**文件类型**：环境变量模板

**职责**：作为"配置说明文档"提交到 Git 仓库。列出项目运行所需的**所有**环境变量名称及其示例值（不含真实密钥）。

包含的变量分组：

| 分组 | 内容 |
|------|------|
| 翻译引擎 | DeepSeek / OpenAI / 百度翻译 的 API Key 和接口地址 |
| 基础设施 | Redis 连接地址、数据库连接字符串（注释状态） |
| 应用行为 | 服务端口、CORS 白名单、调试模式开关 |
| 限流配置 | 每分钟请求上限、每日免费翻译次数上限 |

**与 `.env` 的关系**：`.env.example` 提交到 Git，供所有人查看；`.env` 包含真实密钥，加入 `.gitignore` 永远不提交。每次在新机器上初始化项目时，执行 `copy .env.example .env` 再填入密钥即可。

---

### `.gitignore`

**文件类型**：Git 忽略规则

**职责**：告诉 Git 哪些文件/目录不需要版本控制，防止以下几类内容被误提交：

| 类别 | 忽略内容 | 原因 |
|------|---------|------|
| 环境变量 | `.env`、`.env.local` | 包含真实密钥，绝对不能提交 |
| Python 缓存 | `__pycache__/`、`.venv/`、`*.pyc` | 可在本地重新生成，体积大 |
| Node 依赖 | `node_modules/` | 可通过 `pnpm install` 重新生成 |
| 构建产物 | `.next/`、`dist/`、`.output/`、`*.zip` | 按需本地构建，不需要版本控制 |
| IDE 配置 | `.vscode/`（保留 `settings.json`）、`.idea/` | 个人偏好配置，因人而异 |
| 系统文件 | `.DS_Store`（macOS）、`Thumbs.db`（Windows） | 操作系统自动生成，无意义 |

---

### `README.md`

**文件类型**：项目说明文档

**职责**：项目的"门面"，访问代码仓库时第一眼看到的内容。包含：
- 项目简介和功能定位
- 技术栈一览（前后端、扩展各用了什么）
- 快速开始步骤（安装 → 配置 → 启动）
- 目录结构说明

---

## `shared/` 目录

### `shared/package.json`

**文件类型**：Node.js 子包描述文件

**职责**：让 pnpm 工作区识别 `shared` 是一个可引用的本地包（包名 `shared`）。`packages/web` 和 `packages/extension` 只需在各自的 `package.json` 里添加 `"shared": "workspace:*"` 依赖，就能像导入 npm 包一样直接 `import` 这里的类型，不需要手动维护路径。

---

### `shared/src/index.ts`

**文件类型**：TypeScript 入口文件（当前为空占位）

**职责**：`shared` 包的统一导出入口。目前为空，后续会在这里（或通过子目录）导出前端和扩展共用的内容，例如：
- 翻译请求 / 响应的数据类型定义
- Content Script 与 Background Service Worker 之间的消息通信协议
- 各端共用的工具函数

---

### `shared/tsconfig.json`

**文件类型**：TypeScript 编译配置

**职责**：为 `shared` 子包单独配置 TypeScript 编译选项：

| 配置项 | 值 | 说明 |
|--------|---|------|
| `target` | ES2020 | 编译目标版本，兼容现代浏览器和 Node.js |
| `moduleResolution` | bundler | 适配 Vite / Next.js 等现代打包工具的模块解析方式 |
| `strict` | true | 开启严格类型检查，减少运行时错误 |
| `outDir` | `./dist` | 编译产物输出目录 |

---

## `packages/` 目录占位文件

### `packages/backend/.gitkeep`
### `packages/web/.gitkeep`
### `packages/extension/.gitkeep`
### `docs/.gitkeep`

**文件类型**：空占位文件（命名约定为 `.gitkeep`）

**职责**：Git 不跟踪空目录。在对应子项目尚未初始化之前，用这个空文件确保目录结构被提交到仓库，使其他人 clone 后能看到完整的预期目录骨架，知道后续应该在哪个目录初始化哪个子项目。

等各子项目初始化完成后（Next.js、WXT、FastAPI 都会在各自目录生成真实文件），这些 `.gitkeep` 文件就完成了使命，届时直接删除即可。
