# 前后端联调原理笔记

> 记录日期：2026-03-16

---

## 一、问题背景

前端运行在 `http://localhost:3000`，后端运行在 `http://localhost:8000`，两者是不同的**源（Origin）**。浏览器的同源策略默认会拦截跨域请求，因此需要明确选择一种联调方案。

---

## 二、联调的三种主流方案

### 方案一：后端配置 CORS（本项目采用）

后端直接声明允许哪些域名跨域访问，前端用完整地址发请求。

```python
# FastAPI 示例
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_methods=["*"],
    allow_headers=["*"],
)
```

前端直接请求后端完整地址：

```javascript
fetch("http://localhost:8000/api/v1/translate")
```

**优点**：简单直接，最常用于开发阶段。

---

### 方案二：前端配置代理（Proxy）

前端开发服务器做一层代理，对浏览器隐藏跨域，前端代码只写相对路径。

```javascript
// vite.config.ts
export default {
  server: {
    proxy: {
      "/api": {
        target: "http://localhost:8000",
        changeOrigin: true,
      }
    }
  }
}
```

前端只写相对路径：

```javascript
fetch("/api/v1/translate")  // 实际转发到 http://localhost:8000/api/v1/translate
```

**优点**：前端代码不含硬编码地址，更干净。

---

### 方案三：Nginx 反向代理（接近生产环境）

用 Nginx 统一入口，前后端都走同一域名，彻底消除跨域：

```nginx
location /api/ {
    proxy_pass http://localhost:8000/;
}
location / {
    proxy_pass http://localhost:3000/;
}
```

## 三、本项目（Tom_TranslateApp）采用方案一

### 3.1 后端 CORS 配置

位于 `packages/backend/app/main.py`，FastAPI 应用创建时挂载 CORS 中间件，允许前端和浏览器扩展跨域访问。

### 3.2 前端地址配置

位于 `packages/web/.env.local`：

```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

生产部署时，在 Vercel 后台将此值改为云服务器的真实地址即可，前端代码无需改动。

### 3.3 前后端通信封装

位于 `packages/web/src/lib/api.ts`，这是前后端通信的**核心封装层**，不是配置文件。

**它做了以下事情：**

1. 从环境变量读取后端地址
2. 封装 `translate()` 和 `getLanguages()` 两个函数
3. 前端驼峰命名 → 后端下划线命名（发送时转换）
4. 后端下划线命名 → 前端驼峰命名（接收时转换）
5. 15 秒超时保护（`AbortController`）
6. 统一错误处理（网络错误 / HTTP 错误 / 非 JSON 响应）

**关键代码片段：**

```typescript
const API_BASE = process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000";

// 发请求时做字段命名转换
body: JSON.stringify({
  texts: req.texts,
  source_lang: req.sourceLang,   // 驼峰 → 下划线
  target_lang: req.targetLang,
  engine: req.engine,
})

// 收到响应时做字段命名转换
return {
  translatedTexts: data.translated_texts,  // 下划线 → 驼峰
  sourceLang: data.source_lang,
  engineUsed: data.engine_used,
  tokensUsed: data.tokens_used,
}
```

## 四、完整调用链路

```
用户在页面输入文字
    ↓
TranslatePanel.tsx（UI 组件）调用 translate() 函数
    ↓
api.ts → fetch("http://localhost:8000/api/v1/translate", { method: "POST", body: ... })
    ↓  [跨域由后端 CORS 中间件放行]
FastAPI 后端处理请求，调用翻译引擎
    ↓
后端返回 JSON 响应
    ↓
api.ts 解析响应，转换字段命名
    ↓
TranslatePanel.tsx 渲染译文结果到页面
```

## 五、三个关键文件分工

| 文件 | 角色 | 职责 |
|------|------|------|
| `packages/web/.env.local` | **配置** | 存放后端地址 |
| `packages/web/src/lib/api.ts` | **封装** | 发请求、处理响应、错误处理 |
| `packages/backend/app/main.py` | **后端入口** | 配置 CORS，允许前端跨域 |

**设计原则**：UI 组件不直接写 `fetch`，统一通过 `api.ts` 调用。如果将来后端地址变更、添加鉴权 token，只需修改 `api.ts` 一个文件，所有页面自动受益。

## 六、开发时的典型工作流

1. 启动后端：`uv run uvicorn app.main:app --reload --port 8000`
2. 打开 `http://localhost:8000/docs` 查看接口文档，确认接口正常
3. 启动前端：`pnpm dev:web`（或根目录 `pnpm dev` 同时启动）
4. 打开 `http://localhost:3000` 使用翻译工具
5. 在浏览器 **DevTools → Network 面板** 观察请求和响应
